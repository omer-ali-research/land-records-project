What this is

This is a static dashboard hosted via GitHub Pages. The browser loads summary.json from the same folder and renders tables and totals.

How to update the data

Run python scripts/build_summary.py locally. That script writes data_summary/summary.json (for analysis) and a public-safe copy to docs/summary.json (Drive/Dropbox URLs stripped for the published site).

Parcel data page

The same script scans your local parcel_data folder (see PARCEL_DATA_DIR in scripts/build_summary.py) and writes parcel_counties.json. The published site does not link to cloud storage; the table reflects whatever was baked into the JSON at build time.

Data acquisition page

Row counts come from files under DATA_ROOT in scripts/build_summary.py. The public site omits Google Drive and Dropbox URLs from the JSON copy under docs/; full URLs stay in the private repo / local data_summary/ outputs if you configure them there.

How to turn on the public link (first time only)

If you use a separate public repository for GitHub Pages, enable Pages on that repo (branch main, folder / (root)) and use the Actions workflow in the private repo to deploy. If you publish from this repo’s docs/ folder on main, choose that folder in Pages settings.

  1. Open the public site repository on GitHub → Settings → Pages.
  2. Under Build and deployment → Source, choose Deploy from a branch.
  3. Branch main, folder / (root), then Save.
  4. Open the site URL shown on that page (for example https://YOURUSERNAME.github.io/REPO/).

How to publish updates

Commit and push from the private repo; the Publish public site workflow copies docs/ to the public repo. Or commit docs/ directly to main if Pages reads from this repo.
